# 🐳 Веб-інтерфейс Docker Manager

[Deutsch](README.de.md) | [English](README.md) | [Español](README.es.md) | [Français](README.fr.md) | [Українська](README.uk.md) | [Русский](README.ru.md) | [日本語](README.ja.md) | [中文](README.zh.md)

Потужний, сучасний та футуристичний веб-інтерфейс для управління вашим локальним середовищем Docker. Побудований на Next.js, Tailwind CSS (дизайн Glassmorphism) та Shadcn UI.

![Dashboard Preview](docs/dashboard.png)

## ✨ Можливості

- **Футуристичний UI:** Повністю виконаний у стилі «Glassmorphism», підтримує темний та світлий режими з плавними mesh-градієнтами на фоні.
- **🌍 Багатомовність (i18n):** Інтерфейс повністю доступний **8 мовами**: Українська, Англійська, Німецька, Іспанська, Французька, Російська, Японська та Китайська (Спрощена).
- **🔐 Вбудована автентифікація:** Захищена зона доступу. Програма автоматично створює базу даних SQLite і вимагає входу через (змінні) ім'я користувача та пароль. Стандартний логін: `admin` / `admin`.
- **📦 Управління контейнерами:**
  - Огляд усіх запущених і зупинених контейнерів.
  - Запуск, зупинка, перезапуск або видалення контейнерів.
  - **Термінал у реальному часі (xterm.js):** Відкрийте інтерактивну оболонку прямо у браузері з підтримкою кольорів TTY.
  - **Живі журнали:** Перегляд виводу контейнерів у режимі реального часу.
- **💿 Управління образами:**
  - Список усіх локальних Docker-образів.
  - Видалення окремих образів.
  - Інтелектуальне очищення: видалення зависаючих або невикористовуваних образів одним кліком.
- **🌐 Управління мережами:**
  - Огляд усіх Docker-мереж.
  - Створення нових мереж (опційна підтримка конфігурації підмережі/шлюзу IPv4 та IPv6).
  - Видалення невикористовуваних мереж.
- **🥞 Підтримка стеків (Docker Compose):**
  - Вбудований редактор коду (Monaco) для файлів `docker-compose.yml`.
  - Окрема вкладка для паралельного ведення змінних `.env`.
  - Зручне розгортання (`docker compose up -d`) або зупинка (`docker compose down`) стеків прямо з браузера.

## 🚀 Встановлення та запуск

Програма офіційно **готова до Docker** і включає все необхідне (Alpine Linux, Docker CLI та Node.js-сервер). Вам навіть не потрібно встановлювати Node.js на хост-систему!

### Варіант 1: Запуск через Docker Compose (Рекомендовано)

Готовий шаблон `docker-compose.yml` вже включений. Він монтує локальний Docker-сокет (`/var/run/docker.sock`) у контейнер і надійно зберігає базу даних SQLite та стеки користувача у файловій системі.

```bash
# 1. Клонувати репозиторій
git clone https://github.com/CodeNotiz/docker-manager.git
cd docker-manager

# 2. Зібрати та запустити контейнер у фоновому режимі
docker compose up -d --build
```
Програма тепер доступна за адресою **`http://localhost:3000`**.

### Варіант 2: Класичне налаштування через Node.js (Для розробки)

Передумови: Node.js (v18+) та Docker встановлені на хості.

```bash
npm install
npm run dev
```
Сервер розробки запуститься на порту 3000. *Порада: при першому запуску в фоновому режимі автоматично створюється папка `/data` разом з базою даних SQLite `docker-manager.db`.*

## 🛡️ Автентифікація (Вхід)

Програма захищена Edge Middleware. Без дійсного JWT-куки доступ до панелі керування або API заблокований.

- **Стандартне ім'я користувача:** `admin`
- **Стандартний пароль:** `admin`

*Примітка: Після першого входу настійно рекомендується змінити ці дані!*

### Зміна облікових даних
Натисніть **«Налаштування»** на бічній панелі (внизу ліворуч). Там можна задати нове ім'я користувача та/або пароль. Для підтвердження необхідно ввести поточний пароль (початково `admin`).

## 📁 Структура папок сервера

Окрім коду, сервер під час роботи створює дві важливі директорії в кореневій папці:

- `data/docker-manager.db`: Зберігає ваші облікові дані (паролі шифруються та хешуються через `bcrypt`).
- `stacks_data/`: Якщо ви створюєте Docker Compose Stack у UI, система створює тут підпапку. Вона містить відповідний `docker-compose.yml` та можливий файл `.env`. За потреби ці файли можна редагувати і поза UI.

## 🛠️ Використаний стек технологій

- **Frontend:** Next.js (App Router), React, Tailwind CSS
- **Компоненти:** Shadcn UI, Radix UI, Lucide Icons
- **Термінал і редактор:** xterm.js (включно з WebSockets для PTY), Monaco Editor (`@monaco-editor/react`)
- **Backend (API Routes):** Node.js `fs`, `child_process`, `dockerode` (Docker API Adapter)
- **Автентифікація:** `jose` (JWT), `bcrypt`, `sqlite` / `sqlite3`

## 🤝 Вирішення проблем

* **Відсутні кольори терміналу?**
  Переконайтеся, що при створенні PTY у бекенді середовище налаштовано на `TERM: 'xterm-256color'` (вже інтегровано в код).
* **Попередження Host Next.js (Cross-Origin)?**
  Параметр `allowedDevOrigins` налаштований у `next.config.ts`. Якщо ви розробляєте програму на віддаленому сервері та отримуєте до неї доступ через IP, налаштуйте відповідні IP-адреси у конфігурації.
* **Доступ до `/var/run/docker.sock` заборонено?**
  Ваш виконуючий користувач Node.js має бути членом групи `docker`.
  За потреби запустіть `sudo usermod -aG docker $USER` і увійдіть знову.
